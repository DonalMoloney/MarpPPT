# Creating Marp Decks

This skill encodes how to build a presentation in this repository: write
Markdown, optionally generate chart PNGs from data, and export to PowerPoint.

## When this applies

Trigger on any of:

- "make a deck / presentation / slides / pptx"
- "add a slide about X"
- "turn this CSV/Excel into a chart slide"
- "export to PowerPoint"
- A user opens or edits a `*.md` file with `marp: true` in the front matter.

## Mental model

```
data/*.csv|xlsx ──► make_charts.py ──► figures/*.png
                                             │
                                             ▼
                          slides.md (Markdown + image refs)
                                             │
                                  npx marp slides.md --pptx
                                             ▼
                                        slides.pptx
```

Slide content is plain Markdown. Charts are PNGs generated by a Python
script. Marp CLI converts the Markdown to PowerPoint.

## Workflow

1. **Locate or create the deck.** Decks live next to their assets — see
   `demo/` for the canonical layout. A new deck folder needs at minimum a
   `slides.md`. Optionally add `assets/`, `data/`, `figures/`,
   `make_charts.py`.

2. **Start `slides.md` with the front matter** — exactly this:
   ```markdown
   ---
   marp: true
   theme: default
   paginate: true
   ---
   ```

3. **Write slides.** One idea per slide. Separate slides with `---` on its
   own line. See `docs/writing-slides.md` for layout patterns (backgrounds,
   right-side images, two-column, tables, presenter notes).

4. **If the user gave you data, add a chart slide.**
   - Drop the file into `data/`.
   - Add a function to `make_charts.py` that reads it (`pd.read_csv` or
     `pd.read_excel(..., sheet_name=...)`) and calls `_save(fig, "name")`.
   - Reference the PNG from a slide: `![width:850px](figures/name.png)`.
   - Run `python make_charts.py` and confirm the PNG was written.

5. **Preview** with `npx marp slides.md --preview` while iterating.

6. **Export** with `npx marp slides.md --pptx`. Open the `.pptx` and verify
   slides, images, and the Notes pane before reporting done.

## Hard rules

- Use `--pptx`, NEVER `--pptx-editable` — the editable variant strips
  presenter notes (HTML comments).
- Image paths in `slides.md` are relative to `slides.md`, not the repo root.
  If you move a deck, fix the paths.
- Always re-run `python make_charts.py` after data changes — `npx marp` does
  not regenerate figures.
- Speaker notes are HTML comments inside the slide body: `<!-- ... -->`.
  Don't put them in the front matter.
- Don't hand-edit anything in `figures/`. It's generated output.

## Reference files

- `demo/slides.md` — complete working deck (stock images, table, CSV chart,
  Excel chart, presenter notes).
- `demo/make_charts.py` — complete chart-generation script for both CSV and
  Excel sources.
- `docs/getting-started.md` — install + first-deck walkthrough.
- `docs/writing-slides.md` — layout and image patterns.
- `docs/charts-from-data.md` — full chart workflow.
- `docs/cheatsheet.md` — syntax + CLI reference.

## Verification before claiming done

- `npx marp <deck>.md --pptx` exits 0.
- The `.pptx` file exists and opens.
- If you added a chart slide, the referenced PNG exists in `figures/`.
- If the user mentioned presenter notes, open the `.pptx` and confirm at
  least one Notes-pane entry rendered.
